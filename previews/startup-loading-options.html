CopyShare 启动动画本地预览

这里是本地 HTML 预览，不改正式程序。三个方案都使用当前应用的 Win11 深色基调、蓝色同步高亮和圆角玻璃面板，用于选择启动时最合适的第一印象。

选择方式

打开此文件后观察三组循环动画，然后直接回复“方案一 / 方案二 / 方案三”。确认后再把对应动效接入正式 Vue/Tauri 启动流程。

Option 01

方案一：链路唤醒

推荐主方案
Link wakeup

适合场景

主程序冷启动。先展示两端连接被点亮，再展开主面板，直接表达 CopyShare 的“跨设备同步”核心能力。

优点

识别度最高，和同步产品语义最贴合，启动完成感明确。

取舍

动画元素最多，建议控制在 1.2 秒内，避免显得拖慢启动。

Option 02

方案二：剪贴板流光

更有动感
Clipboard flow

适合场景

强调剪贴板监听和同步通道启动。扫描流光从内容上扫过，表现“正在准备同步”。

优点

动态最明显，用户能感到程序正在加载剪贴板能力。

取舍

更偏功能加载感，品牌记忆点弱于方案一。

Option 03

方案三：极简浮层

最快落地
Minimal overlay

CopyShare

正在准备同步

适合场景

启动耗时很短或希望保持克制时使用。小浮层出现、进度扫过，然后进入主面板。

优点

干净、轻、实现成本低，不会干扰进入主界面。

取舍

仪式感较弱，对“跨设备同步”的表达不如前两个方案直接。

回复“方案一”“方案二”或“方案三”即可。当前默认推荐：方案一：链路唤醒。
